# 1.1 什么是量化投资 - 思想篇

> 完成标志：理解量化投资的基本概念和流程
> 预计学习时间：1-2天

---

## 一、什么是量化投资？

**量化投资 = 用数学模型 + 计算机程序 做投资决策**

简单说：把投资思路写成代码，让电脑帮你做分析 和 交易。

### 对比：主观投资 vs 量化投资

| 主观投资 | 量化投资 |
|---------|---------|
| "我觉得这只股票不错" | "满足A、B、C条件的股票买入" |
| 靠经验和直觉 | 靠数据和模型 |
| 容易受情绪影响（恐慌、贪婪） | 严格执行纪律 |
| 适合研究少数股票 | 可覆盖全市场几千只 |
| 精力有限，容易疲劳 | 24小时运行，不知疲倦 |

## 二、量化投资的基本流程

```
┌─────────────────────────────────────────────────────────────┐
│                     量化投资流程                            │
│                                                             │
│  1. 想法/假设                                               │
│     "低估值股票长期能跑赢市场"                                │
│           ↓                                                  │
│  2. 数据收集                                                │
│     获取历史价格、财务数据                                     │
│           ↓                                                  │
│  3. 策略设计                                                │
│     用代码描述买卖规则                                        │
│           ↓                                                  │
│  4. 回测验证                                                │
│     用历史数据检验策略效果                                    │
│           ↓                                                  │
│  5. 实盘运行                                                │
│     用程序自动/半自动交易                                    │
│           ↓                                                  │
│  6. 监控优化                                                │
│     根据实盘表现调整策略                                      │
└─────────────────────────────────────────────────────────────┘
```

### 每一步详解

#### 1. 想法/假设
一切从想法开始。比如：
- "股价突破20日均线后会继续上涨"
- "低PE的股票长期收益更高"
- "当RSI低于30时是超卖信号"

#### 2. 数据收集
把想法需要的数据准备好：
- 历史价格（开盘价、收盘价、最高价、最低价）
- 财务数据（PE、ROE、营收增速等）
- 宏观数据（可选）

#### 3. 策略设计
用代码把买卖规则写出来：
```python
# 举例：MACD金叉买入策略
if macd > signal_line and previous_macd < previous_signal:
    buy()
```

#### 4. 回测验证
用历史数据模拟交易，看策略表现：
- 收益率多少？
- 最大回撤多大？
- 胜率多少？

#### 5. 实盘运行
用程序真实交易（注意：模拟盘和实盘有巨大差距）

#### 6. 监控优化
根据实盘表现调整策略，但要注意：
> ⚠️ 不要过度优化，否则会过拟合（后面会详细讲）

## 三、散户适合做量化吗？

**适合。原因如下：**

| 优势 | 说明 |
|------|------|
| **门槛低** | Python + 免费数据就能开始，几百块都不用 |
| **情绪可控** | 程序不会恐慌和贪婪，不会因为大跌卖在最低点 |
| **覆盖广** | 可以同时扫描几千只股票，人眼做不到 |
| **纪律严** | 不会因为犹豫错过买卖点 |
| **可回测** | 用历史验证策略再上场，降低试错成本 |
| **适合散户** | 散户资金小、灵活，机构反而不好操作 |

## 四、为什么有人觉得量化是骗人的？

### 常见误区

| 误区 | 真相 |
|------|------|
| "量化能预测市场100%准确" | 量化只能**概率上**有优势，不是100%准确 |
| "回测赚钱就能躺赚" | 过拟合、交易成本、滑点会让实盘亏损 |
| "机构量化都赚钱" | 机构也有亏钱的时候，2022年很多量化基金腰斩 |
| "学量化就能暴富" | 量化是工具，不是印钞机 |

### 机构量化 vs 散户量化

| 维度 | 专业机构 | 散户量化 |
|------|---------|---------|
| 数据 | 毫秒级实时数据，年费百万 | 免费数据（finshare、新浪） |
| 硬件 | 服务器集群，低延迟 | 个人电脑 |
| 团队 | 博士+工程师+金融专家 | 一个人或几个散户 |
| 策略 | 高频、套利、机器学习 | 趋势跟踪、简单指标 |
| 资金 | 百亿+规模 | 几十万-几百万 |

**结论**：机构能做的散户做不了，但散户能做的已经足够赚钱。

## 五、投资方法的完整图谱

### 按实现方式分

| 类型 | 说明 |
|------|------|
| **主观投资** | 人判断、人决策 |
| **量化投资** | 模型驱动、程序执行 |

### 按投资哲学分

| 类型 | 代表人物/方法 |
|------|-------------|
| **价值投资** | 巴菲特：找到好公司，低价买入，长期持有 |
| **成长投资** | 费舍尔：找高增长公司，不在乎估值 |
| **技术分析** | 看K线、均线、形态，不看基本面 |
| **基本面分析** | 看财报、PE、ROE、营收增速 |

### 按管理方式分

| 类型 | 说明 |
|------|------|
| **主动投资** | 想跑赢市场，要选股、择时 |
| **被动投资** | 不跑赢市场，只跟踪指数（ETF定投） |

### 简单粗暴的理解

```
投资方法
├── 主观（人判断）
│   ├── 价值投资（巴菲特那套）
│   ├── 成长投资
│   └── 技术分析（看图炒股）
│
├── 量化（程序执行）
│   ├── 趋势跟踪（MACD金叉什么的）
│   ├── 均值回归（跌多了买，涨多了卖）
│   └── 多因子（PE+ROE+动量一起用）
│
└── 被动（不折腾）
    └── 指数定投（ETF）
```

### 散户常见组合

| 组合 | 特点 |
|------|------|
| 主观价值 + 长期持有 | 选好股拿着不动 |
| 技术分析 + 短线交易 | 看图炒股，追涨杀跌 |
| 量化辅助 + 策略执行 | 用工具辅助决策 |

> ⚠️ 很多人亏钱是因为"混搭"——选了成长股又短线频繁交易，风格不一致很容易亏。

## 八、参考答案

### 第1题参考答案

**常见的散户"经验规则"及代码描述示例：**

| 经验规则 | 代码描述 |
|---------|---------|
| "跌到10块钱就补仓" | `if price <= 10: buy()` |
| "涨了5%就卖" | `if return >= 0.05: sell()` |
| "MACD金叉就买，死叉就卖" | `if macd > signal and prev_macd <= prev_signal: buy()` |
| "放量突破平台就买" | `if volume > ma20_volume and price > resistance: buy()` |
| "PE低于15才考虑买" | `if pe < 15: consider_buy()` |

**关键点**：你所有的经验规则其实都可以用 if-else 描述，这就是量化策略的本质。

---

### 第2题参考答案

**量化能克服情绪问题的原因：**

| 情绪问题 | 量化如何解决 |
|---------|-------------|
| 恐慌（大跌时想卖） | 程序不知道害怕，到止损点才卖 |
| 贪婪（涨了还想涨） | 程序到止盈点自动卖，不贪 |
| 犹豫（该买不买） | 条件满足就买，不拖延 |
| 后悔（卖早了/买晚了） | 严格按照策略执行，不后悔 |
| 疲劳（盯盘累了乱买） | 程序24小时监控，不知疲倦 |

**核心**：人会被情绪左右，程序只认规则。

---

### 第3题参考答案

**机构量化 vs 散户量化的差距：**

| 维度 | 机构量化 | 散户量化 |
|------|---------|---------|
| **数据** | 毫秒级实时数据、逐笔委托、卫星数据、电商数据 | 日线数据为主，免费数据 |
| **硬件** | 服务器集群，托管在交易所旁边，延迟<1ms | 个人电脑，网络延迟几十ms |
| **策略** | 高频交易（秒级）、期权套利、做市商 | 日线级别的趋势跟踪、选股 |
| **团队** | 物理/数学博士 + 工程师 + 金工专家 | 一个人或几个散户 |
| **资金规模** | 百亿到千亿 | 几十万到几百万 |
| **成本** | 数据费百万/年，硬件百万，团队千万/年 | 近零成本 |

**散户学不了的**：
1. 高频交易（延迟和手续费都是致命问题）
2. 另类数据（卫星、信用卡、电商数据，花费巨大）
3. 复杂机器学习（需要数据和算力支持，容易过拟合）

**散户能做的，且够用**：
1. 日线级别的趋势跟踪策略
2. 基于财务数据的多因子选股
3. ETF轮动、资产配置
